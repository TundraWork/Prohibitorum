# v0.5 Downstream SAML 2.0 IdP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Prohibitorum's downstream SAML 2.0 IdP (GHES-compatible profile): SP-initiated SSO (HTTP-Redirect in, HTTP-POST out), IdP-local SLO, IdP metadata, a metadata-ingesting `saml-sp` CLI, stable persistent NameID, the GHES attribute profile, and a first-class XML-security hardening pass — end-to-end smoke-verified against an in-process mock SP.

**Architecture:** Hand-rolled `IdP` in `pkg/protocol/saml` owning the handlers and driving everything from the DB-backed `saml_sp*` schema + the existing session store (Approach B, mirroring the v0.4 OIDC OP). `github.com/crewjam/saml` (SAML message/metadata types) + `github.com/russellhaering/goxmldsig` (XML-DSig sign/verify, C14N) are used as crypto/XML primitives ONLY — never `samlidp.Server`. The same `signing_key` RSA key + x509 cert that signs OIDC also signs SAML.

**Tech Stack:** Go 1.26, `github.com/crewjam/saml` + `github.com/russellhaering/goxmldsig` + `github.com/beevik/etree` (promote to direct), `crypto/rsa`+`crypto/x509`+`encoding/xml`, pgx/v5 via sqlc, chi, the existing KV/session/audit/rate-limiter, and the reused OIDC signing-key infra.

**Spec:** `docs/superpowers/specs/2026-05-30-v0.5-saml-idp-design.md` (decisions D1–D9). **SAML audit reference:** `docs/superpowers/specs/2026-05-24-audit-saml.md`.

**Verification discipline (per `feedback_always_verify_fixes`):** every task lands unit tests; the suite is green at each commit. Task 12 extends `cmd/smoke` to exercise the full v0.5 surface end-to-end against live PG + dev server + in-process mock SP. After all tasks land, a post-implementation parallel audit (crypto/XML-DSig + protocol/standards + race/logic) + a deep second pass (integration/data-integrity/schema-drift), same discipline as v0.3/v0.4 — special focus on XSW/XXE, signature verification, NameID stability, replay.

**Subagent model selection (per `feedback_subagent_model_selection`):** implementers default to **opus**; reviewers may use **sonnet**; never haiku.

**⚠️ LIBRARY-API NOTE:** `crewjam/saml` + `goxmldsig` are NOT yet vendored. Task 0 promotes them. This plan specifies the library ENTRY POINTS (types/functions) and OUR usage precisely, but the EXACT signatures of crewjam/goxmldsig calls MUST be verified by the implementer against the vendored source / pkg.go.dev at implementation time (same as the v0.4 go-jose/v4 API was confirmed at implementation). ## Task 1: Hardened XML/DSig layer (xmlsec.go)

**Goal:** The security-critical primitive layer: parse XML with DTD/XXE disabled; sign an element with RSA-SHA256 + exclusive C14N via goxmldsig; verify a signature against a supplied x509 cert with XSW defenses (the signed `<Reference>` must resolve to the consumed element; reject duplicate IDs; reject SHA-1).

**Files:** Create `pkg/protocol/saml/xmlsec.go`, `pkg/protocol/saml/xmlsec_test.go`

**Acceptance Criteria:**
- [ ] `parseXMLSecure(raw []byte) (*etree.Document, error)` — rejects DTDs / external entities (no XXE); rejects documents with duplicate element IDs.
- [ ] `signElement(doc *etree.Element, key *rsa.PrivateKey, cert []byte) (*etree.Element, error)` — enveloped RSA-SHA256 signature, exclusive C14N, embeds the signing cert in `<X509Certificate>`.
- [ ] `verifyElementSignature(el *etree.Element, cert *x509.Certificate) error` — verifies the enveloped signature against `cert`; rejects SHA-1 digest/signature; confirms the signature `<Reference URI="#id">` resolves to `el`'s own ID (anti-XSW); nil on success, error otherwise.
- [ ] Reuse goxmldsig's `MakeC14N10ExclusiveCanonicalizerWithPrefixList` + `SetSignatureMethod(dsig.RSASHA256SignatureMethod)`.

**Verify:** `mise exec -- go test ./pkg/protocol/saml/... -run TestXMLSec -v -count=1`

**Steps:**
- [ ] **Step 1: `xmlsec.go`.** Use `github.com/beevik/etree` for the DOM and `dsig "github.com/russellhaering/goxmldsig"`. Key structure (verify exact goxmldsig API against the vendored source):
```go
package saml

import (
	"crypto/rsa"
	"crypto/x509"
	"errors"

	"github.com/beevik/etree"
	dsig "github.com/russellhaering/goxmldsig"
	dsigtypes "github.com/russellhaering/goxmldsig/types"
)

var (
	errXMLDTD        = errors.New("saml: XML DTD/entity declarations are not allowed")
	errDuplicateID   = errors.New("saml: duplicate element ID")
	errWeakSigAlg    = errors.New("saml: SHA-1 signatures are rejected")
	errSigRefMismatch = errors.New("saml: signature reference does not cover the processed element")
	errNoSignature   = errors.New("saml: element is not signed")
)

// parseXMLSecure parses raw XML with DTD/external-entity processing OFF and
// rejects duplicate element IDs (anti-XSW). etree does not resolve DTDs by
// default; we additionally scan for <!DOCTYPE/<!ENTITY and reject.
func parseXMLSecure(raw []byte) (*etree.Document, error) {
	doc := etree.NewDocument()
	doc.ReadSettings = etree.ReadSettings{ Permissive: false } // verify field names vs etree version
	if err := doc.ReadFromBytes(raw); err != nil {
		return nil, err
	}
	// Reject DTDs/entities outright (defense beyond etree defaults).
	for _, tok := range doc.Child {
		if d, ok := tok.(*etree.Directive); ok && (containsFold(d.Data, "DOCTYPE") || containsFold(d.Data, "ENTITY")) {
			return nil, errXMLDTD
		}
	}
	if err := assertUniqueIDs(doc.Root()); err != nil { return nil, err }
	return doc, nil
}
```
Implement `assertUniqueIDs` (walk the tree, collect every `ID`/`Id`/`id` attribute, error on a repeat) and `containsFold`. **Verify** the exact `etree.ReadSettings` field names and the goxmldsig signing/validation API against the vendored source.

- [ ] **Step 2: signing.** `signElement` builds a `dsig.SigningContext` with a `dsig.TLSCertKeyStore`/`dsig.RandomKeyStoreForTest`-equivalent that wraps `key`+`cert` (verify the goxmldsig KeyStore interface; the production path supplies our RSA key + DER cert), sets `Canonicalizer = dsig.MakeC14N10ExclusiveCanonicalizerWithPrefixList(nil)` and `SetSignatureMethod(dsig.RSASHA256SignatureMethod)`, then `SignEnveloped(el)` → returns the signed element.

- [ ] **Step 3: verification.** `verifyElementSignature`: build a `dsig.ValidationContext` with a cert store containing ONLY `cert` (`dsig.NewDefaultValidationContext(&dsig.MemoryX509CertificateStore{Roots: []*x509.Certificate{cert}})`); reject if no `<ds:Signature>` child (`errNoSignature`); inspect the `<ds:SignedInfo>` `SignatureMethod`/`DigestMethod` Algorithm and reject any `*sha1*` (`errWeakSigAlg`); read the `<ds:Reference URI>` and confirm it equals `#` + the element's own ID attribute (`errSigRefMismatch`); then `ctx.Validate(el)`.

- [ ] **Step 4: tests (`xmlsec_test.go`, `TestXMLSec...`).** Generate a test RSA key + self-signed cert (reuse `oidc.GenerateSigningKey`-style or `x509.CreateCertificate` inline). Cover:
  - sign an element → verify with the matching cert → success; verify with a DIFFERENT cert → failure.
  - a SHA-1-signed fixture → `errWeakSigAlg`.
  - a doc with two elements sharing an ID → `parseXMLSecure` → `errDuplicateID`.
  - a `<!DOCTYPE>`/`<!ENTITY>`-bearing doc → `errXMLDTD`.
  - a wrapped-signature fixture (signature references element A but element B is the one consumed) → `errSigRefMismatch`.

- [ ] **Step 5: Commit.**

	row, err := i.queries.GetSAMLSubjectID(ctx, db.GetSAMLSubjectIDParams{AccountID: accountID, SpID: spID})
