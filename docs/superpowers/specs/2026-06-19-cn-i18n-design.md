# Simplified Chinese (zh) localization + English copy normalization

**Date:** 2026-06-19
**Status:** Design (approved in brainstorming)

## Goal

Add a complete Simplified Chinese (`zh`) locale to the dashboard SPA, derived from a lightly-normalized English source of truth. The i18n infrastructure is already in place (vue-i18n, a locale-agnostic `LocaleSwitcher`, a `zh` placeholder in `i18n.ts` and the switcher's label map); this fills in the strings, closes two wiring gaps (no persistence, no browser detection), and extends the prod-compiler guard to cover the new locale.

Decisions taken during brainstorming:

- **English scope: light normalization only.** The English copy is already strong and on-voice (PRODUCT.md "calm, exact, welcoming"). Fix consistency issues only; no wording changes.
- **CN register: `你`, pronoun-light.** Warm/modern tone. Drop pronouns where Chinese naturally allows (`管理账户`, not `管理你的账户`); use `你` only when a pronoun is genuinely needed.
- **Term policy: hybrid, leaning technical.** User-facing concepts use precise, technical Chinese terms (e.g. group → `用户组`, not `群组`); protocol acronyms and technical identifiers stay English; admin protocol field labels lean bilingual/English (admins are the technical audience).
- **Locale code: `zh`** (label `中文`), matching the existing infra. Single Simplified variant; no `zh-CN`/`zh-TW` split needed now.
- **Persistence + detection:** persist the chosen locale to `localStorage` and detect `navigator.language` on first load, via a small `useLocale` composable mirroring the existing `useTheme`.

## Non-goals

- No wording/voice rewrite of the English copy (only the mechanical consistency fixes in §1).
- **`time.ts` relative-time units stay English-literal** ("2 hours ago"). Localizing relative time is a separate, deferred effort.
- No `zh-CN`/`zh-TW`/`zh-Hans` locale split or Traditional Chinese.
- No backend changes. (Backend `AuthError` messages are already authored in Chinese; the frontend `errors.*` map provides per-code UI copy and is translated to `zh` here, so `zh` users get Chinese error copy from the same map `en` uses.)
- No change to `LocaleSwitcher.vue` itself (it is already locale-count agnostic and has the `zh → 中文` label).

## 1. English normalization (mechanical, no wording changes)

Applied to `dashboard/src/locales/en.ts`:

1. **Apostrophes → straight ASCII `'`** everywhere. The file currently mixes curly `’` (e.g. `device’s`, `isn’t`, `Don’t`, `I’ve`, `You’ll`, `can’t`, `don’t`) with straight `'` (e.g. `it's`, `You don't`). Standardize to straight `'` for edit-safety and grep-ability (the repo has been bitten twice by the Edit tool flipping straight → curly). Curly double-quotes `“”`, if any, → straight `"` likewise. The ellipsis character `…` (U+2026) is correct and stays.
2. **Ellipsis straggler:** `welcome.fetchingAvatar` `'Setting up your profile picture...'` → `'Setting up your profile picture…'`.
3. **Plural hack:** `admin.account.sessionsRevoked` `'Revoked {count} session(s).'` → `'Revoked {count} sessions.'`.

No keys are added, removed, or reworded. `en.compile.test.ts` must stay green after these edits. (`emailPlaceholder: "name{'@'}example.com"` keeps its `@` escaping — do not touch.)

## 2. Chinese style guide (the translation contract)

### Register & copy rules
- **`你`, pronoun-light.** Omit "you/your" where Chinese reads naturally without it; use `你` (never `您`) when a pronoun is required.
- **Sentence-equivalent casing** — Chinese has no case; keep labels/buttons terse, descriptions as full sentences ending in `。`.
- **Buttons = verb (+ object):** `保存更改`, `删除账户`, `添加通行密钥`. Bare verbs only where context is obvious (`验证`, `继续`).
- **Errors = what happened + what to do, no jargon/blame** (mirrors the English `errors.*` intent). E.g. `rate_limited` → `尝试过于频繁，请稍候再试。`.
- **Preserve interpolation params verbatim:** `{count}`, `{name}`, `{client}`, `{displayName}`, `{date}`, `{when}`, `{app}`, `{idp}`, `{seconds}`, `{n}`, `{username}` appear unchanged inside the Chinese string.
- **Punctuation:** use full-width Chinese punctuation (`，。、：？！`) in prose; keep ASCII for code/identifiers and inside interpolation. No spaces around CJK; a space is fine between CJK and a Latin token (`OIDC 应用`).
- **No literal `@` unescaped** (vue-i18n linked-format) — escape as `{'@'}`; **no literal `|`** (plural separator) or unescaped `{`/`}` in any Chinese string.

## 3. Wiring

### Files
- **Create `dashboard/src/locales/zh.ts`** — mirrors `en.ts` exactly: `export default { … } as const`, identical key tree, every interpolation param preserved.
- **`dashboard/src/i18n.ts`** — import `zh` and add it to `messages` (`messages: { en, zh }`). Remove the deferral TODO comment.
- **Create `dashboard/src/composables/useLocale.ts`** — mirrors `useTheme`:
  - Returns `{ locale, setLocale }` over vue-i18n's global locale.
  - On init: if `localStorage['locale']` holds a registered locale, use it; else detect — `navigator.language?.toLowerCase().startsWith('zh')` → `'zh'`, otherwise `'en'`. Apply the resolved value to the i18n global locale.
  - `watch` the global locale and persist changes to `localStorage['locale']`.
  - Valid-locale guard: ignore a stored value not in the registered locales (fall back to detection).
- **`dashboard/src/App.vue`** — call `useLocale()` once at app root (alongside `useTheme()`), so the persisted/detected locale applies on every route including pre-login pages.
- **`LocaleSwitcher.vue`** — unchanged. Its `v-model="locale"` writes the vue-i18n global locale; `useLocale`'s watcher persists it. The `zh → 中文` label already exists.

### Persistence/detection behavior
- First visit, no stored locale, browser is Chinese → app opens in `zh`.
- First visit, no stored locale, browser non-Chinese → `en`.
- User picks a locale in the switcher → persisted; honored on every later visit regardless of browser language.
- CSP note: like the theme, locale is applied during Vue bootstrap (no inline pre-paint script; `script-src 'self'`). Acceptable; text reflow is instant and pre-auth pages already render after mount.

## 4. Prod-compiler safety + key parity (tests)

- **Extend the compile guard to all locales.** Generalize `dashboard/src/locales/en.compile.test.ts` (rename to `locales.compile.test.ts`, or add a `zh` block) so it compiles **every leaf of both `en` and `zh`** through `@intlify/message-compiler` with a throwing collector — the same path production takes. This catches any unescaped `@`, stray `|`, or unbalanced `{` in the Chinese strings before they blank a subtree in a user's browser.
- **Add a key-parity test** (`locales.parity.test.ts` or a block in the compile test): assert the flattened key set of `zh` is exactly equal to that of `en` (no missing keys → no silent English fallback; no extra keys → no dead strings). This is the guard against translation drift as `en` evolves.

---

## 5. Production method + translation scope

- **What is translated:** every leaf in `zh.ts`, including the full `errors.*` map (so Chinese users get Chinese error copy), admin screens (concepts in Chinese, protocol identifiers per Glossary B), threshold pages, and the account area.
- **Method:** translate **namespace-by-namespace** (each top-level key of `en.ts`: `common`, `admin`, `login`, `errors`, `security`, …) so each batch is independently reviewable and glossary-consistent, followed by a **whole-file consistency pass** (one term, one translation everywhere; param integrity; punctuation; no stray English where Glossary A applies; no over-translation where Glossary B applies).
- **Verbatim-keep inside strings:** CLI commands (`prohibitorum enroll-admin`), code identifiers, URLs/placeholders, and all `{…}` params.

## 6. Testing / gate

- `npx vitest run` green, including the extended compile guard (en + zh) and the key-parity test, and the existing `LocaleSwitcher` test.
- `npx vue-tsc -b` clean (`zh.ts` typed `as const`; structure matches `en`).
- `npm run build` succeeds; rebuild + commit `pkg/webui/dist` at the done-gate (Vite hashes are non-deterministic; dist is committed and embedded via `go:embed`).
- `go build -tags nodynamic ./...` + `go vet ./...` clean (embed compiles).
- A manual spot-check by the operator (native review of the rendered Chinese in both light and dark, across representative member + admin screens and the switcher) is the final acceptance — translation nuance is a human call.
